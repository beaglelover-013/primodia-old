# 普利莫迪亚场景系统备忘录

## 核心原则

不要用穷举玩家自然语言来判断地点。

玩家可能写“回了”“去了”“绕到”“钻进”“跟着某人走”“被带到”“从后门穿过去”“顺路去看一眼”等无限表达。前端不能靠枚举这些说法来改地点。

## 正确分工

- 地点改变只来自权威来源:
  - 前端按钮或地图行动
  - 脚本 action 结算
  - MVU/JSONPatch 中明确写入的地点变量
  - 读档快照中的地点状态
- 玩家自由输入只作为叙事意图:
  - 可以交给 AI 描写
  - 不能由前端自行截词生成地点
- AI 叙事也不是地点权威:
  - AI 说“你回到酒馆”不等于前端地点已改变
  - 除非同一楼层有明确地点变量更新

## 场景状态应通用

不要写成“只管酒馆”或“只管商铺”。

前端应维护一个通用当前场景:

- 大区/城市/地点
- 具体空间
- 当前玩法场景
- 当前临时场景数据
- 在场角色
- 可交互对象

商铺、酒馆厨房、后院、地图城市、人物对话、农田、酒窖，都只是当前场景的不同类型。

## 临时场景清理规则

临时场景必须跟随权威地点变化自动失效。

例如:

- 当前地点不再是商铺时，清空当前商铺和货架缓存。
- 当前地点不再是农田/酒窖时，不把农田临时操作塞进下一回合。
- 当前地点不再是厨房/炉台时，不把上次做菜选择当成下一回合背景。

不能只靠玩家点击“离开”按钮清理；读到变量地点变化时也必须清理。

## 禁止模式

- 禁止从自由文本里抽取“去了X”“回了X”作为地点。
- 禁止用“酒馆/厨房/商铺/街口”等少量词判断所有场景。
- 禁止在地点已变更后保留旧临时场景参与下一次提示词。
- 禁止把“当前页面”当作“当前地点”。

## 推荐实现方向

建立统一的 `sceneContext` 或等价结构:

```ts
type SceneContext = {
  region: string;
  place: string;
  sceneType: 'tavern' | 'shop' | 'farm' | 'cellar' | 'map' | 'character' | 'inventory' | 'custom';
  sceneId?: string;
  transientId?: string;
  presentNpcIds: string[];
  interactables: string[];
};
```

所有页面读取同一个当前场景。所有临时数据挂在 `sceneType/sceneId/transientId` 下。地点改变时，由统一函数清理不再匹配的临时数据。
